GluGo2.0 is a startup comprised of students at Wheaton College, in Norton MA. We build on the work of the machine learning team in GluGo. We're using machine learning to prototype predictive glucose algorithms, the kind that could be implemented in an integrated "closed-loop" system. We seek an algorithm that only utilizes sensor data, without sacrificing the accuracy and personalization of closed-loop algorithms. Such an algorithm could implemented in a sensor-linked alert system. This would sidestep many of the barriers to closed-loop systems by keeping glucose management in the hands of the user, while still providing valuable information about future glucose trends.

Model Building We're currently exploring ARIMA, Markov Chain, and Support Vector Regression models to build an algorithm that can accurately forecast glucose highs and lows at least an hour into the future.

Data Scraping Scraped data from both Medtronic (.csv file from client) & Tidepool (.json file converted to a .csv file) datasets in order to produce an output .csv file containing timestamp, glucose levels (mL/dL), basal insulin (U/hr), insulin sensitivity (mg/dL/U), carb ratio (g/U), bolus insulin and carb input (g). During this process determined algorithms in order to match the carb intake and bolus insulin from the Medtronic data, and the basal insulin, insulin sensitivity and carb ratio from client's Tidepool account to the specific timestamps matching each glucose level read seen in the Tidepool database.
